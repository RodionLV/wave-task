Тестовые задания для кандидата

1. Backend на Go

Задача: написать мини‑сервис управления сетевыми устройствами.

Требования:
- Реализовать REST‑сервис на Go.
- Модель устройства: id, hostname, ip, location, is_active, created_at.
- Эндпоинты:
  - POST /devices – создание устройства.
  - GET /devices – список с фильтрацией по is_active и поиском по hostname (по подстроке).
  - GET /devices/{id} – получение устройства по id.
  - PUT /devices/{id} – обновление устройства.
  - DELETE /devices/{id} – мягкое удаление (пометка флагом, без физического удаления).
- Хранение данных в БД (PostgreSQL, MySQL или SQLite на выбор кандидата).
- Инициализация схемы (миграции или создание таблиц при старте).
- Минимум 2–3 unit‑теста (например, для слоя работы с БД или хендлеров).

Что оцениваем:
- Понимание REST и HTTP.
- Работа с SQL.
- Структура кода, обработка ошибок, тесты.


2. Frontend на TypeScript

Задача: реализовать простую панель управления устройствами.

Требования:
- Single Page Application на TypeScript (React или другой знакомый фреймворк).
- Страница со списком устройств (таблица).
- Функции:
  - просмотр списка устройств;
  - фильтрация по активности (is_active);
  - поиск по hostname;
  - форма добавления/редактирования устройства.
- Чёткие типы для сущностей (модель устройства) в TypeScript.
- Работа с REST‑API (реальный backend из задания 1 или моковый сервер).

Что оцениваем:
- Владение TypeScript/JavaScript.
- Умение работать с REST‑API.
- Структура компонентов и качество кода.


3. SQL и оптимизация запросов

Задача: по предоставленной схеме БД (devices, configs, logs) выполнить несколько подзадач.

Подзадачи:
- Написать запрос для получения:
  - списка активных устройств с количеством связанных configs;
  - последних N записей из logs для конкретного устройства.
- По примеру “тяжёлого” запроса предложить:
  - какие индексы добавить;
  - какие изменения в запросе возможны для ускорения.

Что оцениваем:
- Практическое владение SQL.
- Базовое понимание производительности и индексов.


4. Задача по отладке и анализу ошибок

Задача: дать кандидату небольшой фрагмент кода (на Go или TS) с несколькими ошибками.

Подзадачи для кандидата:
- Найти и описать проблемы:
  - неправильная обработка ошибок;
  - некорректная работа с горутинами/контекстом (для Go);
  - логическая ошибка в условиях или работе с данными;
  - потенциальные утечки ресурсов.
- Предложить исправленный вариант фрагмента кода (можно текстом или патчем).

Что оцениваем:
- Умение читать и разбирать чужой код.
- Навыки дебага и анализа ошибок.


5. Комплексное задание (по желанию)

Задача: мини‑система инвентаризации сетевых устройств.

Требования:
- Backend:
  - Go‑сервис с БД, который хранит устройства и их статусы.
  - REST‑эндпоинты для CRUD‑операций и смены статуса is_active.
  - Логирование основных операций (создание, обновление, удаление).
- Frontend:
  - TypeScript‑клиент, который:
    - показывает список устройств и их статус;
    - позволяет фильтровать и искать устройства;
    - позволяет создавать и редактировать устройства.
- Документация:
  - README с описанием:
    - как запустить backend и frontend;
    - какие зависимости нужны;
    - примеры запросов к API.
- Дополнительно (по желанию кандидата):
  - docker‑compose для запуска БД и сервиса.

Что оцениваем:
- Умение связать backend и frontend.
- Архитектуру приложения в целом.
- Навыки работы с документацией и DevOps‑подходом на базовом уровне.
